# claude-engram demo visual identity

## Style Prompt

Quiet terminal-native product demo for developers evaluating a Claude Code plugin. The visuals should feel like a real terminal session, but edited for clarity: fewer lines, stronger hierarchy, fast payoff, and no decorative marketing panels. The viewer should understand the plugin in the first three seconds.

## Colors

- `#0c0e12` background: near-black terminal canvas
- `#d7dbe0` foreground: primary readable text
- `#7ee787` accent: successful memory injection and "zero effort"
- `#79c0ff` project/path blue
- `#ff9161` Claude/engram mark
- `#1f242c` dividers and terminal chrome

## Typography

- Primary: `JetBrains Mono`, `SF Mono`, Menlo, monospace
- Use larger-than-terminal text where the README thumbnail needs legibility.

## What NOT To Do

- Do not start with empty panels that look broken.
- Do not depend on tiny command output for the core message.
- Do not use floating marketing cards as the primary experience.
- Do not let the payoff arrive after more than three seconds.
- Do not show explanatory preview overlays in rendered GIFs.
